# ⚔️ 環友冒險者公會：新進成員手冊 (Adventurer's Handbook)

歡迎來到 **環友科技 (UTK)** 程式碼戰場！

在這裡，我們不只是在寫 Code，我們是在守護世界、討伐 Bug、並尋找傳說中的寶藏（Features）。本手冊將指引你如何在這個充滿魔法與自動化機器的世界中生存並建立豐功偉業。

---

## 🗺️ 第一章：冒險地圖 (Project Structure)

我們的世界（Repository）由幾個關鍵的魔法視圖組成，請務必將它們加入你的書籤：

1. **⚔️ 目前副本 (Current Sprint Board)**
* 這是你的主戰場。每日站會 (Daily Stand-up) 時我們只看這張地圖。
* **規則：** 保持專注，不要讓太多怪卡在「開發中」。


2. **📜 公會委託書 (Backlog)**
* 所有未來的傳說都在這裡。如果你沒怪打了，或是想知道下一章劇情，請來這裡翻閱。


3. **📚 記錄 (Master Table)**
* 記載一切歷史與數據的總表。如果你想搜尋某個古老的 Bug，請來這裡。

## 🐉 第二章：魔物圖鑑 (Labels & Colors)

在接任務前，你必須學會辨識任務的「顏色」，這決定了你的生存率。

### 1. 威脅等級 (Priority / 優先級)

這是任務的緊急程度，看到紅色請繃緊神經。

| 標籤 | 顏色 | 代號 | 定義與行動準則 |
| --- | --- | --- | --- |
| **P0** | 🔴 Red | **世界毀滅 (World Boss)** | **放下一切，全體動員。** 伺服器失火了，立即停止手邊工作支援前線！ |
| **P1** | 🟠 Orange | **主線劇情 (Main Quest)** | **本次 Sprint 必須修完。** 這是通關條件，沒做完大家都不准下班（誤）。 |
| **P2** | 🟡 Yellow | **支線任務 (Side Quest)** | **排入下次 Sprint。** 重要但現在還能等，有空檔再處理。 |
| **P3** | ⚪ Gray | **彩蛋 (Easter Egg)** | **有資源再說。** 酒館裡的傳聞，甚至可能永遠不修。 |

### 2. 魔物強度 (Complexity / 複雜度)

這決定了你需要消耗多少法力（Story Points）。

| 標籤 | 顏色 | 代號 | 定義與行動準則 |
| --- | --- | --- | --- |
| **XS** | ⚪ Gray | **史萊姆 (Slime)** | **做就對了。** 連討論都不用，一刀解決。 |
| **S** | 🟢 Green | **哥布林 (Goblin)** | **練功機會。** 適合 Junior 工程師累積經驗值。 |
| **M** | 🔵 Blue | **菁英怪 (Elite)** | **主力任務。** 專案的中流砥柱，約佔 Sprint 容量 60%。 |
| **L** | 🌸 Pink | **地下城首領 (Boss)** | **資深追蹤。** 需由 Senior 帶隊，每日站會密切回報血量。 |
| **XL** | 🟣 Purple | **上古巨龍 (Ancient)** | **拒絕開發！** 看到此標籤請勿攻擊。請 PM 先開「拆解會議」將其肢解成小怪。 |

## 🛡️ 第三章：冒險流程 (Workflow)

作為一名合格的冒險者，請遵循以下儀式來完成任務：

### Step 1: 領取委託 (To Do ➝ In Progress)

* 前往 **「⚔️ 目前副本」**。
* 挑選一張卡片，在 **Assignees** 欄位選取自己。
* ✨ **自動化魔法：** 當你指派自己時，卡片會自動飛進 **「🟡 開發中」** 欄位。（若沒反應，請手動拖曳）。

### Step 2: 戰鬥 (Coding)

* 建立分支 (Branch)，開始你的 Coding 之旅。
* 記得定期 Commit，不要讓進度丟失。

### Step 3: 繳交戰利品 (Pull Request)

* 發送 Pull Request (PR)。
* ✨ **自動化魔法：** 你的 PR 會自動出現在看板上，並標示為 **「🟣 審查中」**。
* **重要：** 請在 PR 描述中連結 Issue (例如 `Fixes #123`)，這樣魔法才會生效。

### Step 4: 鑑定與通關 (Review & Merge)

* 邀請同伴進行 Code Review（鑑定裝備）。
* 當 PR 被合併 (Merged) 後，卡片會自動移動到 **「🟢 已完成」**。
* 🎉 **這時，公會的吟遊詩人機器人可能會在 Issue 下方留言慶祝你的勝利！**

## 📜 第四章：如何發布任務 (Issue Templates)

如果你是 PM 或發現了 Bug，請務必使用公會指定的羊皮紙（Template），**禁止使用空白紙條**。

* **📜 發布新委託 (New Quest Spec):**
* PM 專用。
* 必須填寫「勝利條件 (Acceptance Criteria)」與「威脅等級」。
* 若沒填寫清楚，工程師有權拒絕接單。


* **🐞 魔物目擊報告書 (Bug Report):**
* 任何人皆可使用。
* 必須詳細描述「召喚儀式 (重現步驟)」。
* 請附上「留影水晶 (截圖/Log)」，有圖有真相。

## 🤖 第五章：認識你的機器人夥伴

公會裡住了幾隻看不見的精靈（GitHub Actions），請不要被它們嚇到：

1. **公會櫃台 (Receptionist):** 會自動把新開的 Issue 貼到佈告欄，並過濾掉無效的閒聊。
2. **吟遊詩人 (Bard):** 當你關閉 Issue 時，它會隨機出現並給你幾句讚美或虛擬金幣。
3. **瞭望塔哨兵 (Sentry):** 當有人標記 **P0 (世界毀滅)** 時，它會發出全頻廣播警告大家。
